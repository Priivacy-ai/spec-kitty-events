## Summary of Changes

This mission landed the canonical TeamSpace event contract: a single
source-of-truth lane vocabulary that promotes `in_review` to canonical
status (WP01); a structured, machine + human readable `ValidationError`
shape with a closed `ValidationErrorCode` enum (WP02); a recursive
forbidden-key validator that walks every nested object and array
(WP03); reconciled `MissionCreated` / `MissionClosed` /
`WPStatusChanged` payload contracts shared by CLI and SaaS (WP04); a
committed conformance fixture suite organised into the eight named
classes (WP05); a refreshed `COMPATIBILITY.md` distinguishing local
CLI compatibility from TeamSpace ingress validity, plus a 4.x → 5.0.0
schema-version bump (WP06); and finally a per-envelope validation
performance benchmark proving NFR-005's < 5 ms p95 budget along with
this review-handoff document (WP07). All committed work is on the
shared lane workspace `lane-a`, and the full local pipeline (1905
pytest cases, mypy --strict, schema regen) is green at hand-off time.

## Pre-push fixes (2026-05-01, post-merge, pre-`git push`)

A careful pre-push review of the merged mission surfaced five blockers
that were addressed in five follow-up commits on `main` before the
push happened. The original "all green" claim above is now superseded
by the re-captured Verification Results.

1. **Envelope SSOT (issue 1).** The class-taxonomy fixtures used a
   fake envelope shape (`event_type`/`event_version`/`event_id`/
   `occurred_at`/`mission_id`/`payload`) that did NOT match the real
   public `Event` model in `src/spec_kitty_events/models.py`
   (`event_id`/`event_type`/`aggregate_id`/`payload`/`timestamp`/
   `build_id`/`node_id`/`lamport_clock`/`correlation_id`/`project_uuid`/
   `schema_version`). The conformance test wrapper accepted the fake
   shape because it never called `Event.model_validate`. Fixed by
   rewriting every canonical and historical-synthesized fixture to use
   the real Event shape with pinned ULIDs/timestamps; updating the
   test wrapper to call `Event.model_validate` as the public envelope
   SSOT; adding an explicit
   `test_valid_canonical_fixtures_validate_against_real_event_model`
   gate. Invalid fixtures preserved with the same defect so error
   codes still match.

2. **Cutover gate forbidden-key recursion (issue 2).** The public
   `assert_canonical_cutover_signal` only ran the top-level/payload-level
   forbidden-key check, so a `feature_slug` buried inside
   `payload.metadata.tags[2]` silently passed. Fixed by routing
   `cutover.forbidden_legacy_keys` through
   `spec_kitty_events.forbidden_keys.find_forbidden_keys` (the
   recursive walker). Added 6 regression tests proving nested,
   array-element, and depth-bound forbidden keys are now caught while
   string VALUES equal to forbidden names still pass (key-only
   invariant).

3. **Package version vs envelope schema version (issue 3).** Several
   docs (COMPATIBILITY.md, CHANGELOG.md [5.0.0] entry, contracts/
   versioning-and-compatibility.md) claimed the canonical envelope is
   `schema_version="5.0.0"` (or `"4.0.0"` from the prior release). The
   reality: the *package* is `5.0.0` but the *on-wire envelope schema
   version* is `3.0.0` (per `cutover.py::CUTOVER_ARTIFACT.cutover_contract_version`
   and `Event.schema_version` default). The original brief explicitly
   asked for canonical 3.0.0 envelopes. Fixed across all three docs.

4. **Class-taxonomy `min_version` (issue 4).** The 26 class-taxonomy
   manifest entries declared `min_version: "4.1.0"`. They are 5.0.0
   contract additions; bumped to `"5.0.0"`.

5. **Worktree cleanup (issue 5).** Tracked `.gitattributes` (semantic
   merge driver pin from upgrade migration `3.1.1_event_log_merge_driver`)
   in a separate `chore:` commit; removed the orphan
   `.kittify/workspaces/teamspace-event-contract-foundation-01KQHDE4-lane-a.json`
   session-state file; left `.agents/skills/*` intentionally untracked.

The five fixes landed as five commits on `main`:
`a21fcaf` (issue 1) → `5614df1` (issue 2) → `5619d67` (issue 3) →
`ec32328` (issue 4) → `175e2a5` (chore: .gitattributes).

## Open Items / Known Issues

1. **`historical_row_raw` rejection code path.** Today the validator
   used by `tests/test_conformance_classes.py` rejects raw-row fixtures
   via whichever upstream layer (envelope shape, forbidden-key, payload
   shape) trips first, not always with code `RAW_HISTORICAL_ROW`. The
   `expected_error_code` on each manifest entry preserves the *ideal*
   code for the future. The runner accepts any rejection today and
   documents the gap in fixture `notes`. Follow-up WP should add a
   pre-walk "raw-row signature" detector that emits
   `ValidationErrorCode.RAW_HISTORICAL_ROW` first.

2. **Pre-existing mypy warning** in
   `src/spec_kitty_events/conformance/validators.py:395`
   (`unused-ignore` on `import jsonschema  # type: ignore[import-untyped]`).
   Not introduced by this mission; environment-dependent. Recommended
   cleanup is a one-line follow-up; intentionally left untouched here
   to keep WP07's diff confined to `owned_files`.

3. **Python 3.11 vs 3.14 environment.** The lane-a worktree's editable
   install resolves under `python3.11`. Verification commands above
   were run with `python3.11`. The mission does not change supported
   Python versions; `pyproject.toml` already pins `python_version =
   "3.10"` for mypy.
