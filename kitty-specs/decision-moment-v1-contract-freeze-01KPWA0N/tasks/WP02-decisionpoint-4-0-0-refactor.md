# WP02 — DecisionPoint 4.0.0 refactor

## Objective

Execute the core `spec-kitty-events 4.0.0` refactor inside `src/spec_kitty_events/decisionpoint.py`:

1. Introduce the `DecisionPointWidened` event type and `WIDENED` state.
2. Refactor `DecisionPointOpenedPayload`, `DecisionPointDiscussingPayload`, and `DecisionPointResolvedPayload` into Pydantic v2 discriminated unions keyed on `origin_surface ∈ {adr, planning_interview}`.
3. Extend `DecisionPointOverriddenPayload` with an optional `origin_surface`.
4. Extend `ReducedDecisionPointState` with V1 projection fields.
5. Extend `reduce_decision_point_events` to branch on per-event `origin_surface`, idempotently handle duplicate Widened, and detect new anomaly kinds (`origin_mismatch`, `closed_locally_without_widening`).
6. Bump the internal `DECISIONPOINT_SCHEMA_VERSION` to `"3.0.0"`.
7. Land unit coverage in `tests/unit/test_decisionpoint.py` for every variant and transition.

This is the central, highest-risk WP of the mission. Keep ADR semantics under `origin_surface="adr"` byte-identical to today.

## Context

Existing `decisionpoint.py` ships a single `DecisionPointOpenedPayload` with ADR-style required fields (rationale, alternatives_considered, evidence_refs, authority metadata). V1 interview-origin decisions don't have those at ask time — they have question/options/input_key/step_id. The user-approved direction is a discriminated union per event type on `origin_surface`, not a fat conditional payload.

See:
- `spec.md` — FR-001 through FR-014 in full.
- `data-model.md` §2 through §5 — exhaustive field specs and state transition table.
- `research.md` R-1, R-6, R-7, R-8 — rationale.
- `kitty-specs/decision-moment-v1-contract-freeze-01KPWA0N/contracts/` — JSON Schema sketches for every variant.

The existing file is 422 lines. After this WP, expect ~1000–1100 lines.

### T006 — Refactor `DecisionPointOpenedPayload` into discriminated union

**Purpose.** Preserve 3.x ADR semantics exactly while introducing the V1 interview-origin variant without optional-spaghetti.

**Steps.**

1. Import from WP01: `from spec_kitty_events.decision_moment import OriginSurface, OriginFlow`.
2. Add discriminator/union imports: `from typing import Annotated, Union, Literal` and `from pydantic import Field`.
3. Rename the current `DecisionPointOpenedPayload` class to `DecisionPointOpenedAdrPayload`. Add `origin_surface: Literal[OriginSurface.ADR] = Field(...)` as a required field. Keep all existing fields exactly as they are (rationale, alternatives_considered, evidence_refs, authority metadata, identity fields, timestamps).
4. Add a new class `DecisionPointOpenedInterviewPayload` with the interview-origin fields from `data-model.md` §2.2:
   - `origin_surface: Literal[OriginSurface.PLANNING_INTERVIEW]` (required)
   - `decision_point_id`, `mission_id`, `run_id`, `mission_slug`, `mission_type`, `phase` (all `str (min_length=1)`)
   - `origin_flow: OriginFlow` (required)
   - `question: str (min_length=1)`, `options: Tuple[str, ...] = Field(..., min_length=0)`, `input_key: str (min_length=1)`, `step_id: str (min_length=1)`
   - `actor_id: str (min_length=1)`, `actor_type: Literal["human", "llm", "service"]`
   - `state_entered_at: datetime`, `recorded_at: datetime`
   - NO `rationale`, `alternatives_considered`, `evidence_refs`, `authority_role`, `mission_owner_authority_flag`, `mission_owner_authority_path` fields (they're ADR-only; V1 interview Opened is ask-time).
5. Define the union:
   ```python
   DecisionPointOpenedPayload = Annotated[
       Union[DecisionPointOpenedAdrPayload, DecisionPointOpenedInterviewPayload],
       Field(discriminator="origin_surface"),
   ]
   ```
6. Update `_EVENT_TO_PAYLOAD[DECISION_POINT_OPENED]` — it needs to validate against the union. For Pydantic v2, use `TypeAdapter(DecisionPointOpenedPayload).validate_python(payload_dict)` in the reducer. Introduce a helper `_OPENED_ADAPTER = TypeAdapter(DecisionPointOpenedPayload)` at module level.

**Files.**

### T009 — Refactor `DecisionPointResolvedPayload` into discriminated union with cross-field validator

**Purpose.** The most complex payload: enforce `terminal_outcome` and its cross-field constraints (FR-007, FR-008, FR-010, FR-011).

**Steps.**

1. Rename existing class to `DecisionPointResolvedAdrPayload`. Add `origin_surface: Literal[OriginSurface.ADR]` required.
2. Add `DecisionPointResolvedInterviewPayload` per `data-model.md` §2.5 with all V1 fields: `terminal_outcome` (required), `final_answer?`, `other_answer: bool = False`, `rationale?`, `summary: Optional[SummaryBlock]`, `actual_participants: Tuple[ParticipantIdentity, ...] = Field(default_factory=tuple)`, `resolved_by: str (min_length=1)`, `closed_locally_while_widened: bool = False`, `closure_message: Optional[ClosureMessageRef]`, timestamps.
3. Add `@model_validator(mode="after")` on interview variant:
   ```python
   @model_validator(mode="after")
   def _enforce_outcome_fields(self) -> "DecisionPointResolvedInterviewPayload":
       if self.terminal_outcome == TerminalOutcome.RESOLVED:
           if not self.final_answer:
               raise ValueError("final_answer is required when terminal_outcome=resolved")
       else:
           if self.final_answer is not None:
               raise ValueError(
                   f"final_answer must be absent when terminal_outcome={self.terminal_outcome.value}"
               )
           if not self.rationale:
               raise ValueError(
                   f"rationale is required when terminal_outcome={self.terminal_outcome.value}"
               )
           if self.other_answer:
               raise ValueError(
                   f"other_answer must be False when terminal_outcome={self.terminal_outcome.value}"
               )
       return self
   ```
4. Define the union and `_RESOLVED_ADAPTER = TypeAdapter(DecisionPointResolvedPayload)`.
5. Import needed: `TerminalOutcome`, `SummaryBlock`, `ClosureMessageRef` from `decision_moment`, `model_validator` from pydantic.

**Files.** `src/spec_kitty_events/decisionpoint.py` (~150 lines)

### T012 — Extend reducer `reduce_decision_point_events`

**Purpose.** Correctly reduce both ADR and interview-origin event streams, idempotently absorb duplicate Widened, and raise new anomalies for origin mismatch and closed-locally-without-widening (FR-014, replay safety).

**Steps.**

1. Expand the docstring to describe V1 semantics.
2. In the per-event loop, after parsing payload via the appropriate `TypeAdapter`:
   - Track `origin_surface_seen: Optional[OriginSurface] = None` across events for the same `decision_point_id`. If a later event's `origin_surface` differs from the first seen, append `DecisionPointAnomaly(kind="origin_mismatch", event_id=..., message=f"Event origin_surface={new!r} differs from prior {old!r} for decision_point_id={did!r}")`. Still apply the event.
   - Branch projection per event type:
     - **Opened (ADR)**: populate existing 3.x fields as today. Set `origin_surface = OriginSurface.ADR`.
     - **Opened (interview)**: set `origin_surface = OriginSurface.PLANNING_INTERVIEW`, `origin_flow`, `question`, `options`, `input_key`, `step_id`, identity fields. Do NOT populate ADR-only fields (`rationale`, `alternatives_considered`, `evidence_refs`).
     - **Widened**: if `current_state == DecisionPointState.WIDENED` already, no-op (idempotent). Otherwise populate `widening: WideningProjection(...)` from the payload and transition to `WIDENED`. Do not clear prior fields.
     - **Discussing (ADR)**: populate rationale/alternatives/evidence as today.
     - **Discussing (interview)**: populate `last_actor_id`, `last_actor_type`, `last_state_entered_at`, timestamps. Do not touch ADR fields. Optional: expose snapshot as a new optional projection field (skip for V1; keep scope minimal).
     - **Resolved (ADR)**: populate rationale/alternatives/evidence as today.
     - **Resolved (interview)**: populate `terminal_outcome`, `final_answer`, `other_answer`, `summary`, `actual_participants`, `resolved_by`, `closed_locally_while_widened`, `closure_message`. If `closed_locally_while_widened=True` and `widening is None` (no prior Widened for this `decision_point_id`), append `DecisionPointAnomaly(kind="invalid_transition", message="closed_locally_while_widened=True without prior DecisionPointWidened")` AND skip projecting that field (leave `closed_locally_while_widened=False` on the reduced state).
     - **Overridden**: unchanged ADR-style projection.
3. Duplicate Widened handling: before calling `_ALLOWED_TRANSITIONS`, if `target_state == DecisionPointState.WIDENED and current_state == DecisionPointState.WIDENED`, `continue` without appending an anomaly. Count it in `event_count` (already counted post-dedup; no change).
4. Update `DecisionPointAnomaly.kind` docstring to enumerate new valid values: `"origin_mismatch"` (added).
5. Authority/LLM policy checks: apply only to ADR variant (existing behavior). Interview variant has no authority_role field; skip those checks for interview variant.
6. Final freeze: build `ReducedDecisionPointState` with all projected fields. Ensure default values are used for absent projections (e.g., interview-origin stream never populates `last_rationale`).

## Risks

- **Pydantic v2 discriminator syntax.** Double-check with a small reproducible example before committing. `Annotated[Union[...], Field(discriminator="origin_surface")]` must work inside a `TypeAdapter`.
- **Forward references.** If `WideningProjection` is defined in `decision_moment.py` and uses `ParticipantIdentity`, ensure the circular import edge is cleanly broken. If the chain gets ugly, move `WideningProjection` to `decisionpoint.py` instead.
- **Reducer regression.** Any behaviour change to existing ADR reducer semantics breaks downstream consumers. Run the existing `test_decisionpoint_reducer.py` and `test_decisionpoint_determinism.py` suites before adding any V1 logic — they must pass.
- **Event counting.** Idempotent Widened must NOT double-count `event_count` (it's counted post-dedup in step 3 of reducer; that's unaffected).
- **TypeAdapter performance.** Instantiate adapters once at module level (module-level `_OPENED_ADAPTER = TypeAdapter(...)`), not per event.
