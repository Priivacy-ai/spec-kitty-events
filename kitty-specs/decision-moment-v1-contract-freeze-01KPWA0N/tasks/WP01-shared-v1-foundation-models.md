# WP01 — Shared V1 foundation models

## Objective

Land the reusable foundation for the `spec-kitty-events 4.0.0` Decision Moment V1 contract:

1. Create a new module `src/spec_kitty_events/decision_moment.py` holding V1 enums and shared Pydantic models that every subsequent WP imports.
2. Extend `ParticipantIdentity` in `src/spec_kitty_events/collaboration.py` with an optional `external_refs` block, preserving full 3.x compatibility.
3. Lock both additions with unit tests under `tests/unit/`.

This WP has no dependencies. It is the foundation every other WP (especially WP02) compiles against.

## Context

The `spec-kitty-events 4.0.0` contract freeze introduces Decision Moment V1. Several cross-cutting shared models are used by both the new `DecisionPointWidened` event and the refactored `DecisionPointResolved` payload (interview variant):

- A canonical participant-identity extension with optional Slack/Teamspace references (FR-012, FR-013).
- A structured `SummaryBlock` for resolved decisions with provenance (FR-009).
- Typed references for Slack threads, teamspaces, channels, and closure messages (FR-005, FR-010, FR-011).
- Shared enums (`OriginSurface`, `OriginFlow`, `TerminalOutcome`, `SummarySource`, `WideningChannel`, `DiscussingSnapshotKind`).

See the mission's `data-model.md` §1 and §2.1 for exhaustive field definitions and `research.md` R-2 and R-5 for rationale.

### T002 — Extend `collaboration.py` with `ParticipantExternalRefs` and `ParticipantIdentity.external_refs`

**Purpose.** Carry optional Slack/Teamspace identity alongside the canonical `participant_id` so replay and downstream rendering don't need live lookups (FR-012, FR-013).

**Steps.**

1. In `src/spec_kitty_events/collaboration.py`, near the existing `ParticipantIdentity` (around lines 54–70), add `ParticipantExternalRefs`:
   ```python
   class ParticipantExternalRefs(BaseModel):
       """Optional cross-channel identity references for a ParticipantIdentity."""
       model_config = ConfigDict(frozen=True, extra="forbid")

       slack_user_id: Optional[str] = Field(None, min_length=1)
       slack_team_id: Optional[str] = Field(None, min_length=1)
       teamspace_member_id: Optional[str] = Field(None, min_length=1)

       @model_validator(mode="after")
       def _at_least_one(self) -> "ParticipantExternalRefs":
           if not any((self.slack_user_id, self.slack_team_id, self.teamspace_member_id)):
               raise ValueError(
                   "ParticipantExternalRefs must populate at least one of "
                   "slack_user_id, slack_team_id, teamspace_member_id"
               )
           return self
   ```
2. Extend `ParticipantIdentity` with `external_refs: Optional[ParticipantExternalRefs] = None`. Keep all existing fields unchanged. Keep `model_config = ConfigDict(frozen=True)` (no `extra="forbid"` — it was already permissive).
3. Add `model_validator` import from pydantic at the top of the file if not already imported.
4. Confirm `from spec_kitty_events.collaboration import ParticipantExternalRefs` works.
5. Do not modify any existing test or consumer of `ParticipantIdentity`.

**Files.**
