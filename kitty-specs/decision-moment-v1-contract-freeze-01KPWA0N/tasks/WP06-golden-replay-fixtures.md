# WP06 — Golden replay fixtures and integration replay tests

## Objective

Ship 6 golden replay fixture pairs (`.jsonl` event streams + paired `_output.json` reduced-state snapshots) covering every V1 scenario, mirror them into the package's conformance tree so downstream consumers can depend on them, and wire `tests/integration/test_lifecycle_replay.py` to replay each pair and assert byte-identical output.

## Context

Golden replay fixtures are the ultimate contract lock. A downstream consumer — `spec-kitty#757` CLI, `spec-kitty-saas#110` projection, etc. — can pick up any of these fixtures, replay through the 4.0.0 reducer, and get the exact output. If the reducer drifts, the test fails.

See:
- `spec.md` FR-016, NFR-001, Scenarios 1–5 + edge cases.
- `data-model.md` §8.3 — exhaustive event sequences.
- `research.md` R-3 — rationale.

### T022 — Create 6 golden replay fixture pairs (plus conformance mirrors)

**Purpose.** The canonical replay truth for V1 Decision Moments.

**Steps.**

1. For each scenario in `data-model.md` §8.3, write:
   - A `.jsonl` file under `tests/fixtures/decisionpoint_golden/<name>.jsonl` containing one JSON-encoded `Event` per line in Lamport-ordered sequence. Use the event envelope shape the repo already ships (consult any existing `.jsonl` under `tests/fixtures/decisionpoint_golden/` — the `replay_full_lifecycle.jsonl` in the repo is the reference).
   - A `.json` file `<name>_output.json` containing the expected reduced state, dumped via `json.dumps(ReducedDecisionPointState(...).model_dump(mode="json", by_alias=True), sort_keys=True, indent=2)`. Indent for readability is fine — the test must compare using the *same* dump settings on both sides.
2. Use stable ULIDs, Lamport clocks starting at 1, and fixed timestamps (`2026-04-23T10:00:00+00:00`, `10:00:01`, etc.) so the fixtures are byte-deterministic.
3. Per-scenario event sequences:
   - `replay_interview_local_only_resolved` — two events: `DecisionPointOpened(interview)`, `DecisionPointResolved(interview, terminal=resolved, final_answer="oauth2", closed_locally_while_widened=false)`.
   - `replay_interview_widened_resolved` — four events: `DecisionPointOpened(interview)`, `DecisionPointWidened`, `DecisionPointDiscussing(interview, snapshot_kind="digest", contributions=["...","..."])`, `DecisionPointResolved(interview, terminal=resolved, summary.source="slack_extraction", summary.candidate_answer="...", actual_participants=[...with external_refs], closure_message populated)`.
   - `replay_interview_widened_closed_locally` — three events: `Opened`, `Widened`, `Resolved(terminal=resolved, closed_locally_while_widened=true, summary.source="mission_owner_override", closure_message populated)`. Actual_participants may be empty.
   - `replay_interview_deferred` — two events: `Opened`, `Resolved(terminal=deferred, rationale="need security review", no final_answer)`. Do NOT include Widened here; keep scenario minimal.
   - `replay_interview_canceled` — two events: `Opened`, `Resolved(terminal=canceled, rationale="out of scope", no final_answer)`.
   - `replay_interview_resolved_other` — two events: `Opened(options=["session","oauth2","oidc","Other"])`, `Resolved(terminal=resolved, final_answer="internal SSO proxy", other_answer=true)`.
4. Mirror every pair into `src/spec_kitty_events/conformance/fixtures/decisionpoint/replay/` with identical contents so the package ships them as public conformance artifacts. `pyproject.toml` `package_data` already covers this subtree.

### T023 — Extend `tests/integration/test_lifecycle_replay.py` for V1 golden replays

**Purpose.** Automated replay assertion: every golden pair reproduces its output byte-identically via `reduce_decision_point_events`.

**Steps.**

1. Open `tests/integration/test_lifecycle_replay.py`. Preserve existing tests.
2. Add a parameterized test:
   ```python
   V1_GOLDENS = [
       "replay_interview_local_only_resolved",
       "replay_interview_widened_resolved",
       "replay_interview_widened_closed_locally",
       "replay_interview_deferred",
       "replay_interview_canceled",
       "replay_interview_resolved_other",
   ]

   @pytest.mark.parametrize("name", V1_GOLDENS)
   def test_v1_golden_replay_byte_identical(name: str) -> None:
       fx = pathlib.Path("tests/fixtures/decisionpoint_golden")
       events_file = fx / f"{name}.jsonl"
       expected_file = fx / f"{name}_output.json"
       events = [Event.model_validate_json(line)
                 for line in events_file.read_text().splitlines()
                 if line.strip()]
       reduced = reduce_decision_point_events(events)
       actual = json.dumps(
           reduced.model_dump(mode="json", by_alias=True),
           sort_keys=True, indent=2,
       )
       expected = expected_file.read_text().rstrip()
       assert actual == expected, f"Replay drift in {name}"
   ```
3. Factor shared helpers (`_load_events`, `_dump_state`) if helpful.

**Files.** `tests/integration/test_lifecycle_replay.py` (extend ~80 lines)

**Validation.**
- [ ] `pytest tests/integration/test_lifecycle_replay.py -q` green.
- [ ] Tampering with any `_output.json` causes the matching test to fail with a clear diff.

## Risks

- **Byte-identical output is fragile.** Any drift in Pydantic's serialization (e.g., key ordering, optional field emission) breaks goldens. Pin dump settings: `model_dump(mode="json", by_alias=True)` + `json.dumps(..., sort_keys=True, indent=2)`. Do not use `exclude_none=True` unless explicitly required (optional fields emitted as `null` are fine if the goldens reflect that).
- **Timezone representation.** ISO-8601 UTC strings: use `+00:00` not `Z` for consistency with the rest of the repo's fixtures.
- **Tuple vs list in reduced state.** `Tuple[ParticipantIdentity, ...]` serializes as a JSON array. Confirm the _output.json shows arrays, not strings.
- **Existing conformance replay fixtures.** The repo already has goldens like `replay_full_lifecycle.jsonl` in `tests/fixtures/decisionpoint_golden/`. Do NOT modify them; add new files only.

## Reviewer guidance

- Open each `_output.json` and verify field presence/absence matches the scenario — e.g., deferred fixture has no `final_answer`, no `summary`, but has `terminal_outcome: "deferred"` and a non-null `last_rationale`/`rationale`.
- Verify `closure_message` is populated on `widened_closed_locally_output.json` and `widened_resolved_output.json`.
- Verify `actual_participants` carries `external_refs.slack_user_id` on the `widened_resolved` fixture.
- Verify each `.jsonl` is sorted by Lamport clock on disk (for readability, even though the reducer re-sorts).
