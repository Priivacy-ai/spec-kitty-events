# WP03 — Reducer extension tests and property determinism

## Objective

Prove the V1 reducer extensions from WP02 are correct and deterministic:

1. Extend the reducer unit test suite with V1 transition cases, anomaly-kind cases, and projection-field assertions.
2. Extend the property/determinism test suite with Hypothesis strategies for interview-origin events, Widened, and Other/free-text resolution.
3. Lock byte-identical replay under any Lamport-consistent permutation (NFR-001, NFR-005).

## Context

WP02 refactors `decisionpoint.py` with discriminated-union payloads, a new Widened event, an extended state machine, and new anomaly kinds (`origin_mismatch`, `invalid_transition` for `closed_locally_while_widened` without prior Widened). The reducer must be deterministic and replay-safe; this WP codifies that.

Background:
- `spec.md` FR-014, NFR-001, NFR-005, edge-case table.
- `data-model.md` §4 transition table, §5 projection fields.
- `research.md` R-8 reducer scope.

Hypothesis is already a dev dependency (see `pyproject.toml`). The existing `tests/property/test_decisionpoint_determinism.py` is the reference for style and ≥500-run settings.

### T013 — Extend `tests/test_decisionpoint_reducer.py` with V1 transitions and anomaly cases

**Purpose.** Nail down every named transition and anomaly kind with a dedicated unit test.

**Steps.**

1. Open `tests/test_decisionpoint_reducer.py`. Preserve all existing tests unchanged (they cover ADR semantics and must still pass).
2. Add a new section `# V1 reducer coverage (Decision Moment V1)` with these tests (naming follows existing convention):
   - `test_reducer_opened_interview_only_projects_ask_time_state` — single interview Opened event. Assert `state=OPEN`, `origin_surface=PLANNING_INTERVIEW`, `origin_flow=<expected>`, `question`, `options`, `input_key`, `step_id` populated; ADR fields `last_rationale`, `last_alternatives_considered`, `last_evidence_refs` are `None`.
   - `test_reducer_opened_interview_then_resolved_resolved` — Opened → Resolved(resolved, final_answer="X"). Assert `state=RESOLVED`, `terminal_outcome=RESOLVED`, `final_answer="X"`, `other_answer=False`, no anomalies.
   - `test_reducer_opened_interview_then_resolved_deferred` — Opened → Resolved(deferred, rationale="Y"). Assert `state=RESOLVED`, `terminal_outcome=DEFERRED`, `final_answer is None`, `other_answer=False`.
   - `test_reducer_opened_interview_then_resolved_canceled` — same shape, canceled.
   - `test_reducer_opened_interview_then_widened_resolved_resolved` — Opened → Widened → Resolved(resolved). Assert `widening` projection populated with channel/teamspace_ref/thread_ref/invited_participants. State goes `OPEN → WIDENED → RESOLVED`. Actual_participants populated from Resolved.
   - `test_reducer_widened_then_discussing_then_resolved` — full Slack discussion flow. `state = DISCUSSING` between Widened and Resolved events (if Discussing appears); final `state=RESOLVED`.
   - `test_reducer_duplicate_widened_is_idempotent` — Opened → Widened → Widened → Resolved. Assert exactly one `widening` projection (second Widened is absorbed), no anomalies raised, `state=RESOLVED` at end.
   - `test_reducer_closed_locally_while_widened_true_sets_field` — Opened → Widened → Resolved(resolved, closed_locally_while_widened=True). Assert `closed_locally_while_widened=True` in reduced state, no anomaly.

### T014 — Extend property tests with V1 Hypothesis generators

**Purpose.** Generate thousands of synthetic interview-origin streams to stress-test the reducer for non-determinism and transition errors.

**Steps.**

1. Open `tests/property/test_decisionpoint_determinism.py`. Preserve existing ADR strategies and tests.
2. Add new Hypothesis strategies (prefix `st_`, composed via `st.builds` where possible):
   - `st_origin_flow()` — `st.sampled_from(list(OriginFlow))`.
   - `st_terminal_outcome()` — `st.sampled_from(list(TerminalOutcome))`.
   - `st_summary_source()` — `st.sampled_from(list(SummarySource))`.
   - `st_summary_block(source: Optional[SummarySource] = None)` — returns a `SummaryBlock` dict with `text`, `source`, optional `extracted_at`, optional `candidate_answer`.
   - `st_participant_identity()` — returns a `ParticipantIdentity` dict with `participant_id`, `participant_type`, sometimes `external_refs`.
   - `st_interview_opened_payload(decision_point_id)` — full interview Opened payload dict.
   - `st_widened_payload(decision_point_id)` — full Widened payload dict.
   - `st_interview_resolved_payload(decision_point_id, terminal: TerminalOutcome, *, widened: bool = False)` — terminal-respecting interview Resolved payload dict (enforces cross-field constraints at generation time).
   - `st_interview_stream(decision_point_id)` — composite strategy that emits one of: `[opened, resolved]`, `[opened, widened, resolved]`, `[opened, widened, discussing, resolved]`, `[opened, widened, widened, resolved]` (idempotency), with random terminal_outcome.
3. Add new property tests:
   - `test_interview_reducer_deterministic_across_runs` — `@given(events=st_interview_stream(...))`, `@settings(max_examples=500, deadline=None)`. Reduce twice; assert equal output.
   - `test_interview_reducer_handles_any_valid_terminal_outcome` — generate streams with each terminal_outcome; assert reducer projects `terminal_outcome` matches the last Resolved event's value.
   - `test_interview_idempotent_widened_never_double_projects` — generate streams with random duplicate Widened counts (2–5); assert `widening` is projected exactly once and matches the first Widened's content.
4. Ensure `@settings(max_examples=500, deadline=None)` is applied to each property test.

### T015 — Assert byte-identical replay under permuted Lamport orderings

**Purpose.** Lock NFR-001 at the property-test level: no matter how a replay consumer orders events (within Lamport constraints), the reduced output must be byte-identical.

**Steps.**

1. Add a new property test in `tests/property/test_decisionpoint_determinism.py`:
   - `test_decisionpoint_reducer_byte_identical_under_permutation`.
   - `@given(events=st_interview_stream(...))`, `@settings(max_examples=500, deadline=None)`.
   - Helper `_dump_state(state)` → `json.dumps(state.model_dump(mode="json", by_alias=True), sort_keys=True, separators=(",", ":"))`.
   - Generate events, compute baseline dump.
   - Use Hypothesis `permutations` strategy to generate a permutation of the events list. Filter permutations that violate Lamport ordering (or rely on the reducer's internal `status_event_sort_key` to normalize; then any permutation is fine).
   - Reduce the permuted events; compute permuted dump.
   - Assert baseline == permuted.
2. Document in a test comment that determinism is enforced by `status_event_sort_key` + `dedup_events` at the start of the reducer pipeline.

## Risks

- **Hypothesis flakiness.** If Hypothesis finds a seed that produces non-deterministic output, investigate the reducer (likely a projection field is populated from a non-deterministic source, e.g., Python dict ordering before 3.7 — should not be an issue on 3.10+). Do NOT weaken the test; fix the reducer via a WP02 follow-up.
- **Performance of 500 max_examples per test.** On 3 property tests × 500 = 1500 runs, suite wall time should remain <30 s. If slower, tune down to 300 but never below.
- **Permutation space explosion.** With 10+ events, permutations become unwieldy. Cap event count per stream strategy at ~6 events to keep Hypothesis shrinking tractable.

## Reviewer guidance

- Confirm each new test has a descriptive name referencing the FR/NFR or behavior it enforces.
- Confirm property tests use `@settings(max_examples=500, deadline=None)`; anything less fails the charter Testing gate.
- Confirm `st_interview_stream` never generates a stream that would cause the cross-field validator to reject (e.g., `terminal_outcome=deferred` with `final_answer` set). Otherwise the reducer will never see those invalid events and the test becomes about schema rejection, not reducer determinism.
